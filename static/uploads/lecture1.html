Precision and recall

  • So now that we have this general matrix, we can define a few general metrics as well.
  • How can we distill the information from the confusion matrix into a single number?
  • Recall: where missing positives is costly.
  • \[ \small \text{precision} = \frac{TP}{TP+FP} = \frac{\text{Correctly\ positive\ predictions}}{\text{All\ positive\ predictions}} \]
  • \[ \small \text{recall} = \frac{TP}{TP+FN} = \frac{Correctly\ positive\ predictions}{All\ actual\ positives} \]

Optimize for precision or recall?

  • Optimize precision = TP / (TP + FP)
    • Minimize FP: make few positive predictions
  • Optimize recall = TP / (TP + FN)
    • Minimize FN: predict all samples to be positive.
  • F1-score is the harmonic mean of precision and recall: \[ f1 = 2 \times \frac{ \text{precision} \times \text{recall}}{\text{precision} + \text{recall}}\]
  • Optimize precision
    • Example: make a single positive prediction and it is correct.
  • It is important to consider both precision and recall, but how do we adjust how much of each we get?
  • F1 as a harmonic mean of precision and recall, which could be used in hyperparameter optimization.
  • The harmonic mean is used instead of the arithmetic mean because it gives more weight to lower values. If either precision or recall is very low, the F1 score will be low, reflecting that the models performance is not good enough in one aspect.

How are we making predictions?

  • What happens to precision and recall if we change the probability threshold?
  • How do we choose the threshold?

  • Lgistic regression gives probability as an outcome, use the threshold of 0.5 to make decision.

  • What happens to precision and recall if we change the probability to a lower number like 0.2?

  • Decreasing the threshold:

    • Increases the number of positive predictions
    • Identify more examples as fraud examples
    • More false positives, more true positives.
    • Recall would either stay the same or go up and precision is likely to go down (precision may increase if all the new examples after decreasing the threshold are TPs).

  • Increasing the threshold:

    • Decreases the number of positive predictions
    • Recall would go down or stay the same but precision is likely to go up (precision may go down if TP decrease but FP do not decrease).

  • Key question: How do we decide on the best threshold? It depends on the problem and trade-offs between false positives and false negatives.

Precision-recall curve: precision and recall as the threshold varies

  • We can look at all possible thresholds and plot the corresponding precision and recall!

  • As we lower the threshold, the classifier predicts more positives:

    • Recall increases because we capture more true positives.
    • Precision usually decreases because some of the additional positives are false positives.

  • As we raise the threshold, the classifier predicts fewer positives:

    • Precision increases because only the most confident predictions are positive.
    • Recall decreases because we miss more true positives.

  • The curve shows the trade-offs between precision and recall across all thresholds.

  • Ideal performance: top-right corner (high precision and high recall).

  • Use cases like fraud detection often require focusing on areas of the curve that favor one metric over the other (e.g., high recall for safety-critical tasks).

Average precision (AP)

  • AP summarizes the PR curve by calculating the area under the curve.

  • AP computes the weighted mean of precisions achieved at each threshold, with the increase in recall from the previous threshold used as the weight (the area under the PR curve).

  • AP score has a value between 0 (worst) and 1 (best).

  • A high AP score indicates that the classifier performs well across different thresholds.

  • In this example, we compare Logistic Regression (LR) and Support Vector Classifier (SVC).

  • We can see that the SVC classifer performs slighlty better for most thresholds, which means that its AP score is slightly higher, as we can see in the chart.

ROC curve

  • False positive rate (FPR) and true positive rate (TPR or recall).

\[ TPR = \frac{TP}{TP + FN}\] \[ FPR = \frac{FP}{FP + TN}\]

Area under the ROC curve (AUC)

  • AUC measures the ability to rank positive instances higher than negative ones.
  • AUC = 1 means a perfect model.
  • A random guessing model has AUC = 0.5.
  • Whats the probability that a randomly picked positive point has a higher score according to the classifier than a randomly picked point from the negative class.
  • AUC of 1.0 means all positive points have a higher score than all negative points.
  • For classification problems with imbalanced classes, using AP score or AUC is often much more meaningful than using accuracy.
  • AUC evaluates the models ranking capability across all thresholds.

Class imbalance

  • The training set has significantly more examples of one class.
  • Handling imbalance:
    • Change the weights of classes.
    • Oversample minority class or undersample majority class.

Handling imbalance by chaning class weights

  • We can specify class_weight="balanced" (weight each class inversely proportional to their frequency.)
  • This sets the weights so that the classes are equal.
  • We have reduced false negatives but we have many more false positives now
    • Decreases false negatives, which improves recall.
    • Increases false positives, which might lower precision.
