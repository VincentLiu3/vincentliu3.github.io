Lecture 8: Model interpretability

Steps for building a ML model with tabular data

  • Understanding the problem
  • EDA, preprocessing
  1. Train-test split. Don’t look at the test set until the end.
  2. EDA (class imbalance, outliers, collinearity, …)
  3. Choose a scoring metric depends on the problem.
  4. Feature engineering.
  5. Create a preprocessing pipeline.
  • Model building
  1. Train a baseline model (e.g., dummy classifier).
  2. Try a linear model.
  3. Try other models (e.g. LightGBM).
  4. Pick a few reasonable models. CV score is a reasonable metric, but you may choose simpler models.
  5. Hyperparameter optimization.
  6. Try averaging and stacking.
  • Model interpretation
  1. (Optional) Some diagnostics like confusion matrix or residual plots.
  2. Model interpretation: feature importance or SHAP plots.
  3. Test set evaluation.
  4. Summarize your results and discuss them with stakeholders.

Tree-based feature importance

How can we get feature importances for tree-based models?

Two ways to get feature importances

  • Algorithm dependent: e.g., calculated based on the information given by the model algorithm (e.g., impurity or entropy).
  • Model agnostic: e.g., measuring increase in prediction error after permuting feature values.

Feature importance as decrease in impurity

  • Feature importance for feature \(j\) = the total decrease in impurity (or entropy) for all splits using feature \(j\).
  • Normalized across all features to sum to 1.
  • For ensemble methods, we average over all trees.

Pros and cons

  • Simple, fast and convenient.


  • Can only be used for tree-based models.
  • Computed on the training data.
  • The importance can be high even for features that are not important to predict the target values.
    • High importance of high cardinality features.
    • Correlated features share importance.
  • We don’t know if features contribute positively or negatively.
  • We don’t know what contributed to individual predictions E.g. “why was this person denied a loan”
  • Measures what was useful for the tree.

Permutation importances

  • Randomly shuffling the values of a single feature and observing the change in the model’s score.
  • A feature that is useful for predictions must have some systematic relationship to the class labels
    • It can be a complicated non-linear relationship, but there must be some pattern that the model can pick up on.
  • We could measure the usefulness of a feature by destroying this relationship and observe how the model performance changes
    • E.g. Permuting (shuffling) the data in one column at a time, and measuring model performance
  • Somewhat similar to forward and backward selection
    • But since we don’t drop the column completely, we don’t have to refit the model, which is a big win in terms of speed
    • We probably have also already done feature selection at this point.

Permutation importance algorithm

  1. Fit a model on the training data.
  2. Compute the reference score \(s\) on the validation data.
  3. For each feature \(j\):
  • Randomly shuffle column \(j\).
  • Compute the score \(s_{k,j}\) with the permuted data.
  • The importance for feature \(j\) is the change in score: \[\small s - \frac{1}{K} \sum_{i=1}^K s_{k,j}\]
  • No need to refit the model so this is quite fast!
  • E.g. a drop in accuracy of 15%.

Shap (SHapley Additive exPlanations)

  • Explain the prediction of an example by computing the contribution of each feature to the prediction.

How to explain a specific prediction?

  • Start at a base rate (e.g., how often people get their loans rejected).
  • We want to approximate how the prediction changes in the presence of a feature with all possible combinations of the other features.
  • Instead we try to approximate this scenario by observing how the predictions change when we replace a feature value with different random numbers while keeping other features constant.
  • This allows us to compute how each feature contributes to changing the prediction away from the base rate/expected value, E[f(x)] below.
  • For each prediction, SHAP values quantify how much each feature contributes to moving the prediction from the base value (mean prediction) to the actual prediction.
  • Uses concepts from cooperative game theory.

Shap values

  • Shap values indicate the contribution of each feature value to the final prediction.
  • The same unit as the target:
    • For classification, it is a probability or log odds depending on the model.

Pros and cons

  • We measure what matters (model performance)
  • We use validation/test data instead of train data.
  • Model agnostic.
  • Don’t inflate values of high cardinality features
  • We know if features contribute positively or negatively
  • Can say what contributed to individual predictions E.g. “why was this person denied a loan”


  • Correlated features are still an issue for most explainers (TreeExplainer is supposed to handle correlations).

One big caveat

  • Feature importance to the model is not the same as feature importance to the problem in the real world.
  • Feature importances explain how a model works, not how the world works.
  • This is true for all types of feature importances.
  • Features that have low importance for a bad model (low cross-validation score) could be very important for a good model.
  • Therefore it is always important to evaluate the predictive power of a model using a held-out set (or better with cross-validation) to interpret importances.
  • Feature importances from a good model are a better indicator of what could be important in the real world and they could inform what would be interesting to look into in the real work, but we should stay away from making causal claims about the world based solely on feature importances.
