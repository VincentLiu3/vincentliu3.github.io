Data cleaning

  • We must have good data to build a good model.
    • garbage in -> garbage out!
  • Examples of low quality data:
    • Out of bounds values (e.g. proportions over 1).
    • Using multiple values to represent the same value (e.g. NA, NONE, NULL, NaN).
    • Duplicate observations
    • Untidy format
  • Data cleaning = fixing mistakes in the data.

However, we can’t fix all issues with data cleaning

  • Many missing value
  • Very few observations of some specific groups
  • Biased sample that is not representative of the population
  • Non-independent observations presented as if they are independent
  • Many more

Feature preprocessing and feature engineering

  • Preprocessing includes things such as imputations and scaling features.
  • What does feature engineering mean? How is it different from feature preprocessing?
    • Designing new useful features.
  • Preprocessing includes things such as imputations and scaling features, which we have alredy talked about previously.
  • Preprocessing tend to focus on things you “should” do and without it your model might either run into an error (e.g. not knowing how to handle a missing value) or do something your don’t intend it to (e.g. KNN on non-standardized data).
  • Feature engineering is the process of designing new features that we think can be informative to the model, by combining the existing features (or features from alternative data sources) in different ways.
  • Feature engineering can be seen as more optional in the sense that it involves operations that likely will improve your model.
  • However, this is not a hard distinction and we could refer to cleaning, preprocessing, and engineering all together as “feature preprocessing” which you will see e.g. in sklearn.

Guidelines for designing new features

  1. Is not unique or random
  • Not useful: Phone number, SIN, etc
  1. Has variance (is not constant)
  • Not useful: Anything that is the same for all observations. For example, same home state, insurance provider ,etc

Guidelines for designing new features

  1. Adds unique variance (is not a constant transformation)
  • Not useful: Weight in lbs, height in feet, age + 10
  1. Is interpretable
  • Not useful: \(\text{BMI} = \sin(\text{Weight}^2) / \sqrt{\text{Height}}\)
