Ensemble methods

  • Combining individual models improves performance.
  • We could combine any model, but often use decision trees because they are interpretable.
  • Most winning solutions in Kaggle competitions use ensembles.
  • Key factors for effective ensembles:
    • Expertise (good performance for each model)
    • Diversity (different types of errors)
  • Expertise: Each model performs reasonably well on its own.
  • Diversity: Models should make different types of errors for averaging to be effective.

Expertise: our models only need to be better than random guessing

Diversity

  • Key Idea: Diversity ensures that models make different types of errors, which can be averaged out in the ensemble to improve performance.
  • How can we encourage our diversity?
    • Bagging.
    • Boosting.

Bagging

Bagging - Bootstrap aggregation

  • Key Idea: Train each model on a bootstrap sample (random sample with replacement) instead of the full training set.
  • More diversity because each model use different data.

Random forest (RF)

  1. Data diversity: Each decision tree is trained on a bootstrap sample of the data.
  2. Features diversity: At each split, only a random subset of features is used.

Random Forest extends bagging by introducing additional randomness: Data randomness: Bootstrap sampling. Feature randomness: Random subset of features at each split. This additional randomness reduces correlation between trees, leading to a lower variance ensemble. While this may increase bias slightly, the reduction in variance typically outweighs the bias, resulting in better generalization. Random Forest is superior to basic bagging because it achieves greater diversity among individual models while maintaining strong predictive power.

Random forest in sklearn

from sklearn.ensemble import RandomForestClassifier

clf = RandomForestClassifier(n_estimators=100, max_depth=None, max_features='sqrt')
  • n_estimators: number of decision trees (higher = more stable predictions).
  • max_depth: max depth of each decision tree (higher = more model complexity).
  • max_features: the number of features at each split (higher = more complexity).

Extremely randomized trees: ExtraTrees

  • Instead of finding the best split, ExtraTrees choose random thresholds for splits.
  • Individual tree is very weak.
  • RF and ExtraTrees often perform better than bagging decision tree alone.
  • The choice of features in ExtraTrees is still based on the same process as in Random Forests, but the thresholds for splitting within the selected features are randomized.
  • Unlike Random Forests, ExtraTrees do not use bootstrap sampling by default. Instead, they train each tree on the full dataset.

Why bagging works?

  • Bagging averages high-variance models.
  • By averaging predictions, uncorrelated errors cancel out, lowering the overall variance and reducing overfitting.

Which of these four scenarios represents a random forest (each square is a tree)?

  • Each tree is often grown to full depth, so they will tend to overfit (high variance, low bias).
  • Averaging many high variance/low bias trees yields precise and accurate predictions.

Advantages of Random Forest

  • Each tree is trained independently, making Random Forests easy to parallelize.
  • Tend to have good performance with default params:
    • The more estimators the better for prediction, but slower.
    • At some point test performance will plateau but it usually does not fall, even when training performance increases.
    • Tuning Tips: at least max_features (and class_weight for imbalanced data).

Boosting

  • Boosting combines many simple models (weak learners) to create a strong learner.
  • Models (e.g., trees) are built sequentially, where each subsequent tree focuses on correcting the errors of the previous ones.

Adaptive boosting (Adaboost)

  • Start by fitting a classifier on the training data.
  • Assign higher weights to examples that the classifier got wrong. (weights grow exponentially each time when it is wrong).
  • Fit a new classifier on the weighted training data.
  • Repeat until the max number of estimators is reached.
  • Final prediction: majority/weighted voting.
  • Good, but usually worse performance than gradient boosted trees…

Gradient boosting: improves a weak tree by adding multiple other trees

  • Construct a base tree (e.g., a tree with just the root node).
  • Build next tree to predict the residuals (errors) of the previous tree.
  • Combine the current tree with the previous ones.
  • Repeat until max num of estimators is reached.
  • Final prediction: use the additive tree to make predictions (no voting).

Summary

  • RF: Many independent trees with random diversity and high variance.
  • Boost: Many dependent trees with non-random diversity and high bias.
  • Gradient boosted tree tends to have better performance than bagging-based trees.

  • The main idea in gradient boosting is that we are trying to decrease the residuals from the previous tree, by creating a new tree that can predict the residuals.

  • In contrast to keep splitting the same tree, each new tree in boosting will see try to model the residuals from the first data to split the entire data again rather than continuing with the subset of data existing after the first split

  • Gradient boosted tree tends to have better predictive performance than bagging-based trees, but can be more complex to interpret.

Stacking

  • Instead of averaging the outputs of each estimator, stacking uses the outputs as inputs to another model.

Stacking

  • For classification:
    • A simpler model, like logistic regression, is often used as the meta-model.
    • The features for the meta-model are the outputs of the base classifiers (e.g., probabilities or predictions), not the original features.
  • Stacking = weighted voting: The meta-model learns a weight for each of the input models.
  • Pass the predict_proba from each of the original models to the meta-model, which then uses these to predict y
  • Each model could be an ensemble
  • In effect, this becomes a weighted voting, where the meta-model associates a weight which each of the input models depending on how accurate they are on the training data I.e. each model is a feature
  • The number of coefficients = the number of base estimators!
